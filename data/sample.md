# RAG 示例知识库

本文档用于演示从零实现的检索增强生成（RAG）系统。

## 项目简介

本项目是一个教学向的 RAG 流水线，包含以下步骤：

1. **文档加载**：从 `data/` 目录读取 txt、md、pdf 文件
2. **文本分块**：按字符数与重叠窗口切分，保留来源元数据
3. **向量嵌入**：通过本地 Ollama 调用嵌入模型（默认 `nomic-embed-text`）
4. **向量检索**：使用余弦相似度在本地索引中查找 Top-K 相关片段
5. **答案生成**：将检索到的上下文拼入提示词，由 Ollama 对话模型生成回答

## 技术栈

- **嵌入与生成**：Ollama（本地运行，无需云端 API Key）
- **向量存储**：NumPy + JSON 持久化到 `storage/` 目录
- **Web 界面**：Streamlit

## 默认配置

| 参数 | 默认值 |
|------|--------|
| 分块大小 | 500 字符 |
| 分块重叠 | 80 字符 |
| Top-K 检索 | 4 |
| 嵌入模型 | nomic-embed-text |
| 对话模型 | qwen2.5:7b |

## 注意事项

- 运行前请确保 Ollama 已启动，并已 pull 嵌入模型与对话模型
- 扫描版 PDF 无 OCR 时检索效果较差，建议使用文本型 PDF
- 更换嵌入模型后需使用 `python ingest.py --rebuild` 重建索引
